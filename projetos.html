problema

Acoplamento com gateway único.

O SolarZ Pay — plataforma B2B de cobranças e assinaturas em crescimento — operava com forte dependência de um único provedor de pagamento. Trocar de gateway exigiria mexer no core financeiro, com risco operacional e comercial.

solução

Camada de integração agnóstica.

No time de engenharia, sob revisão contínua do tech lead, implementei uma camada de abstração aplicando Design Patterns:

  • Interface única de cobrança consumida pelo domínio
  • Implementações isoladas para ASAAS e IUGU (Strategy + Adapter)
  • Clean Architecture + SOLID para isolar regras de domínio
  • Dashboards Next.js para operação dos clientes B2B
resultado

Alternância de provedor sem tocar no core.

A plataforma transaciona R$ 2M de MRR em 271 clientes empresariais com infraestrutura desacoplada. Alternar provedor passou a ser decisão de produto, não projeto de engenharia.

problema

Clínicas dispersas em planilhas.

Psicólogos autônomos gerenciavam atendimentos, prontuários e cobranças em ferramentas fragmentadas (planilha, Google Agenda, WhatsApp solto), sem conformidade adequada com LGPD para dados sensíveis de saúde.

solução

SaaS de gestão de clínica + IA.

Como co-fundador técnico, defini stack, padrões de design e modelagem de domínio:

  • Backend Spring Boot: agendamento, prontuário, billing
  • DDD aplicado nos módulos de prontuário e agendamento
  • Autenticação Spring Security + OAuth2 + Keycloak
  • Automação WhatsApp com LLMs p/ relacionamento
  • Pipeline CI/CD em VPS + Docker Hub + GitHub Actions
resultado

3+ anos no ar, sem DevOps dedicado.

Plataforma em operação contínua há mais de 3 anos, atendendo 164 psicólogos ativos. Stack mantida por mim como co-fundador técnico — produto, infraestrutura e conformidade LGPD sem precisar de especialista DevOps no estágio inicial.

FinCat // projeto pessoal · ia aplicada · 2025→
estudo pessoal spring boot + llms whatsapp api react + typescript vps + docker
// natureza
Projeto pessoal
// canal
WhatsApp
// extração
LLM
// infra
VPS + CI/CD
problema

IA generativa no ecossistema Spring.

Queria explorar na prática como integrar LLMs a fluxos de domínio em Spring Boot — saindo do nível de demo para algo com pipeline real, mantido em produção.

solução

Motor de extração via WhatsApp.

Projeto pessoal onde o usuário registra fluxos financeiros em linguagem natural via WhatsApp, e a IA estrutura o dado:

  • Motor de extração com LLM em Spring Boot
  • Identifica valor, categoria e forma de pagamento
  • Converte dados não-estruturados em registros organizados
  • Clean Architecture + SOLID no backend
  • CI/CD próprio em VPS — baixo custo operacional
resultado

Aprendizado em produção, não em demo.

FinCat virou laboratório vivo de IA aplicada — LLMs integrados a fluxo de domínio real, pipeline próprio, deploy contínuo. O que aprendo aqui carrego para os outros sistemas.

problema

Saúde pública precisa de UX.

Profissionais de saúde nos estados de RN e ES operavam sistemas governamentais densos, com alta carga cognitiva, em alta demanda (campanhas de vacinação, gestão hospitalar).

solução

Front-end no Módulo Farmácia.

Desenvolvimento front-end do Módulo Farmácia (Gestão Hospitalar):

  • Arquitetura React/Vue para fluxos complexos de inventário
  • Otimização da interface de vacinação
  • Foco em redução de carga cognitiva
  • Refinamento técnico com stakeholders governamentais
resultado

Impacto em escala pública.

Interface usada por profissionais de saúde de dois estados brasileiros em fluxos críticos de vacinação e gestão hospitalar. Aprendizado sobre código escalável em equipes multidisciplinares e Scrum aplicado a software de governo.
